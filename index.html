CLI for persistent wiki-based knowledge systems

Turn source documents into a growing markdown wiki.

`wiki-builder` ingests documents with an LLM, writes structured markdown pages, maintains cross-links, and lets you query accumulated knowledge instead of re-deriving context from scratch every time.

Read the docs Get started

Why it exists

Most document tools answer questions. This one builds memory.

Persistent synthesis

Each ingest updates a real wiki with source summaries, concept pages, entity pages, and an evolving index.

Markdown-first output

The knowledge base lives in normal files you can open in Obsidian, VS Code, or any text editor.

Agentic workflow

The LLM uses file tools to read the schema, inspect the wiki, and update multiple pages in a single operation.

How it works

Documents go in. Structured knowledge comes out.

1

Create a wiki

Initialize a project with `schema.md`, `raw/`, and a writable `wiki/` directory.

2

Add source files

Drop markdown, text, or converted PDF content into `raw/`.

3

Ingest with the LLM

The agent creates summaries, concepts, entities, index updates, and log entries.

4

Query and compound

Ask questions against the wiki and optionally save strong answers back into it.

Project layout

The wiki stays transparent and inspectable.

raw/                  # immutable source documents
├── paper.md
├── article.md
└── assets/

wiki/                 # LLM-maintained markdown knowledge base
├── index.md
├── log.md
├── sources/
├── concepts/
└── entities/

schema.md             # conventions and workflows the LLM follows

Quickstart

Go from empty folder to searchable wiki in minutes.

git clone https://github.com/yourname/wiki-builder
cd wiki-builder
pip install -e .

wiki init my-research/ --name "AI Research"
cp ~/papers/attention-is-all-you-need.md my-research/raw/
cd my-research
wiki ingest raw/attention-is-all-you-need.md
wiki query "what is self-attention and why does it matter?"
wiki lint
wiki status
